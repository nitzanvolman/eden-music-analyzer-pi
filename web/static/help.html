Analysis Reference

Each feature extracted by the analyzer, its OSC output, and what it means for your light show.

Time-Domain Features

Amplitude /audio/amplitude 30 Hz

Tracks the volume envelope of the audio signal. Amplitude.kr follows volume increases and decreases with configurable attack and release times.

Output: Single float, 0.0 (silence) to ~1.0 (full scale).

Use case: Drive overall brightness, scale effects, or trigger level-based thresholds.

Config: SC_AMP_ATTACK_TIME (snappiness), SC_AMP_RELEASE_TIME (decay speed).

Pitch /audio/pitch 30 Hz

Detects the fundamental frequency of the audio using SuperCollider's Pitch.kr algorithm. Returns both the detected frequency and a confidence score.

Output: Two floats — frequency in Hz (e.g. 440.0 for concert A) and hasFreq (0.0 = no pitch, 1.0 = confident).

Use case: Map pitch to color hue, or use confidence to distinguish pitched (melodic) vs unpitched (percussive) content.

Config: SC_PITCH_MIN_FREQ (default 50 Hz), SC_PITCH_MAX_FREQ (default 4000 Hz). Narrow for vocals (80-1000), widen for bass (30+).

Spectral Features (FFT-based)

Loudness /audio/loudness 30 Hz

Perceptual loudness in sones, using a psychoacoustic model. Unlike raw amplitude, this accounts for how humans actually hear — we're more sensitive to midrange frequencies.

Output: Single float, 0 (silence) to ~64+ (very loud).

Use case: More natural-feeling brightness control than raw amplitude. Better for music with wide dynamic range.

Spectral Centroid /audio/spectral/centroid 30 Hz

The "center of mass" of the frequency spectrum — a measure of perceived brightness or timbral warmth. High values mean bright/trebly sound, low values mean dark/bassy.

Output: Single float, frequency in Hz (typically 200–8000 for music).

Use case: Map to color temperature — low centroid = warm reds/oranges, high centroid = cool blues/whites.

Spectral Flatness /audio/spectral/flatness 30 Hz

Measures how noise-like vs tone-like the sound is. Pure tones (flute, sine wave) have low flatness. Noisy sounds (drums, static, crowd noise) have high flatness.

Output: Single float, 0.0 (pure tone) to 1.0 (white noise).

Use case: Distinguish rhythmic/percussive sections from melodic sections. Drive chaos/order visual parameters.

Onset Detection Suite event

Ten parallel onset detectors, each tuned for a distinct musical event using different algorithms and frequency bands. All fire independently as trigger pulses.

Output: Each channel sends 1 on trigger. Event-driven, not periodic.

Use case: Flash lights on kicks, trigger strobes on snares, sync visuals to melodic note changes — each independently routable.

Percussive channels (mkl algorithm):

/audio/onset/kick — Kick drum (LPF 100 Hz) | /audio/onset/snare — Snare, claps (BPF 200–2000 Hz) | /audio/onset/hihat — Hi-hats, cymbals (HPF 5000 Hz) | /audio/onset/perc — Any percussive hit (full spectrum)

Melodic channels (phase algorithm):

/audio/onset/bass — Bass note changes (LPF 200 Hz) | /audio/onset/melody — Vocal/synth notes (BPF 300–4000 Hz) | /audio/onset/bright — High-pitched changes (HPF 3000 Hz)

General channels:

/audio/onset/any — Catch-all (rcomplex) | /audio/onset/drop — Big energy spikes (power) | /audio/onset/soft — Only obvious onsets (rcomplex, high threshold)

Config: SC_FEATURE_ONSET_SUITE (master toggle), SC_ONSET_{NAME} (per-channel on/off), SC_ONSET_{NAME}_THRESHOLD (per-channel sensitivity).

Timbral & Harmonic Features

MFCC /audio/mfcc 10 Hz

Mel-Frequency Cepstral Coefficients — a compact "fingerprint" of the sound's texture/timbre. Used in speech recognition and music classification. Each coefficient captures a different aspect of the spectral shape.

Output: Array of floats (default 13 coefficients). Coefficient 0 ≈ overall energy. Coefficients 1–5 = broad spectral shape. Higher = finer detail.

Use case: Feed into pattern matching for genre/mood detection. Visualize as a "timbre barcode" that changes character with the music.

Config: SC_MFCC_NUM_COEFF (default 13, reduce to 7 to save CPU, up to 42 for maximum detail).

Key Detection /audio/key 2 Hz

Estimates the musical key by analyzing pitch class distribution (chroma) and matching against key profiles. Returns a pitch class (C through B) and mode (major or minor).

Output: Two integers — key (0–11: C, C#, D, ... B) and mode (1 = major, 0 = minor).

Use case: Set color palette based on key (each key gets a signature color). Switch between warm (major) and cool (minor) palettes.

Config: SC_KEY_DECAY_TIME (default 2.0s). Higher = more stable, lower = faster key changes. Note: KeyTrack works best with FFT size 4096.

Chromagram /audio/chroma 10 Hz

Energy distribution across the 12 musical pitch classes (C, C#, D, ... B) regardless of octave. Shows "which notes are playing right now" — think of it as a real-time chord/harmony detector.

Output: Array of 12 floats — energy per pitch class, ordered C through B.

Use case: Map each note to a color on a 12-point wheel. Visualize chord progressions. Detect harmonic content for reactive lighting.

Note: Requires Chromagram UGen from SCMIRUGens (included in redFrik standalone, must be built separately for from-source installs).

Rhythm Features

Beat Tracking /audio/beat + /audio/bpm event

BeatTrack2 detects beat positions and estimates tempo. Uses template-matching across straight and swung rhythmic patterns. The beat trigger fires on each detected beat; BPM is sent alongside.

Output: /audio/beat sends 1 on each beat (event-driven). /audio/bpm sends estimated tempo in BPM (float).

Use case: Sync strobes/pulses to the beat. Use BPM to set animation speeds. The core timing signal for any beat-reactive light show.

Config: SC_BEAT_LOCK — 0 = free (adapts quickly to tempo changes, good for live DJs), 1 = locked (holds steady tempo, good for produced tracks).

Scene & Energy Features

Scene Change Detection /audio/scene_change event

Fires a trigger when the spectral content changes significantly — breakdowns, drops, track transitions, filter sweeps. Uses PV_HainsworthFoote spectral novelty detector.

Output: Sends 1 on each detected scene change (event-driven).

Use case: Trigger dramatic lighting changes on drops and transitions. Flash entire rig on major spectral shifts.

Config: SC_SCENE_THRESHOLD (default 1.0, higher = fewer, more dramatic triggers). Toggle: SC_FEATURE_SCENE.

Energy Direction /audio/energy_direction 10 Hz

Compares short-term energy to long-term energy. Detects whether the music is building up (approaching a drop) or breaking down (entering a breakdown). Near zero means steady energy.

Output: Single float, -1.0 (breaking down) through 0.0 (steady) to +1.0 (building up).

Use case: Gradually intensify lighting as energy builds, dim or cool down during breakdowns. Drive anticipation effects before drops.

Config: SC_ENERGY_FAST_RELEASE (default 0.5s, short-term envelope), SC_ENERGY_SLOW_RELEASE (default 8.0s, long-term envelope). Toggle: SC_FEATURE_ENERGY.

Vocal Likelihood /audio/vocal 10 Hz

Heuristic vocal detector. Band-passes to the vocal range, then combines pitch confidence (tonal content), inverse spectral flatness (not noise), and amplitude gating into a likelihood score. Also triggers on tonal synth leads — acceptable for EDM lighting use.

Output: Single float, 0.0 (no vocal/tonal content) to 1.0 (strong vocal/tonal presence).

Use case: Highlight vocals with spotlight effects. Distinguish vocal sections from instrumental drops for different lighting moods.

Config: SC_VOCAL_MIN_FREQ (default 300 Hz), SC_VOCAL_MAX_FREQ (default 4000 Hz). Toggle: SC_FEATURE_VOCAL.

Input Conditioning

Signal Chain

Optional signal processing applied before all analysis: Gain → High-Pass Filter → Noise Gate → Compressor. All stages are transparent at default values — existing behavior is unchanged unless you set them. Tune for live recordings with ambient noise (wind, crowd, generators).

Gain: SC_INPUT_GAIN (default 1.0). Normalize quiet or hot recordings so thresholds work consistently.

High-Pass Filter: SC_HPF_FREQ (default 20 Hz). Set to 30–40 Hz for wind/generator rumble on playa.

Noise Gate: SC_NOISE_GATE_THRESHOLD (default -96 dB, effectively off). Set to -40 to -30 for noisy environments. Prevents false triggers during breakdowns.

Compressor: SC_COMPRESSOR_ENABLED (default 0). Tames dynamic range so quiet and loud sections analyze consistently. SC_COMPRESSOR_THRESHOLD, SC_COMPRESSOR_RATIO, SC_COMPRESSOR_ATTACK, SC_COMPRESSOR_RELEASE.

Configuration

Trigger Rates

Features are grouped by how often they send data:

Fast (default 30 Hz): amplitude, pitch, loudness, centroid, flatness — responsive features for real-time reactivity.

Medium (default 10 Hz): MFCC, chromagram — computationally heavier, lower rate is sufficient.

Slow (default 2 Hz): key detection — musical key changes slowly, high rate would be noisy.

Event-driven: onset, beat — fire only when detected, not on a timer.

Config: SC_TRIG_RATE_FAST, SC_TRIG_RATE_MEDIUM, SC_TRIG_RATE_SLOW. Lower rates save CPU and network bandwidth.

Feature Toggles

Each feature can be individually disabled to save CPU on the Raspberry Pi. Disabled features are completely excluded from the analysis — no UGen computation, no OSC messages sent.

Config: SC_FEATURE_AMPLITUDE, SC_FEATURE_PITCH, SC_FEATURE_LOUDNESS, SC_FEATURE_CENTROID, SC_FEATURE_FLATNESS, SC_FEATURE_ONSET_SUITE, SC_FEATURE_MFCC, SC_FEATURE_KEY, SC_FEATURE_BEAT, SC_FEATURE_CHROMA, SC_FEATURE_SCENE, SC_FEATURE_ENERGY, SC_FEATURE_VOCAL (all default 1 = enabled, set to 0 to disable). Individual onset channels have their own toggles (SC_ONSET_{NAME}).
